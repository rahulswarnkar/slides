Overview

Syntax/Semantics

- Uses white space to delimit code blocks - Semicolons after statements are optional 

		def foo(x):
			if x == 0:
				bar()
				baz()
			else:
				qux(x)
				foo(x - 1)
## Indentation is very important ```python try: x = 1 while True: print(x) x+=1 except KeyboardInterrupt: pass ```
## Indentation in REPL ```shell >>> def foo(x): ... if x == 0: ... bar() ... baz() ... else: ... qux(x) ... foo(x - 1) ... >>> ```

Strongly, Dynamically Typed

- **Strong**: the type of a value doesn't change in unexpected ways. - **Dynamic**: runtime objects (values) have a type, as opposed to static typing where variables have a type. 

>>> x = [1, 2, 3]
>>> type(x)
<class 'list'>
>>> y = 'Hello!'
>>> type(y)
<class 'str'>
>>> z = 42
>>> type(z)
<class 'int'>
>>> x = 1.0
>>> type(x)
<class 'float'>
>>> y = {}
>>> type(y)
<class 'dict'>
>>> z = True
>>> type (z)
<class 'bool'>
## Duck Typing > If it walks like a duck and it quacks like a duck, then it must be a duck.
An object's suitability is determined at runtime by the presence of certain methods and properties, rather than the type of the object itself
